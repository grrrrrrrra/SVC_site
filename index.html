Abstract

Figure 1: Visualization of the instruction ambiguity.

Natural language instructions in human–robot interaction often contain subtle ambiguities that hinder reliable interpretation. These arise when a single instruction can be understood in multiple ways, assigning conflicting semantic roles to objects, tools, or participants.

To address this, we propose Semantic Valency Conflict (SVC) — a cognitively inspired, logit-free method for detecting ambiguity in robot-directed instructions. SVC identifies divergences in role assignments across alternative interpretations of a predicate using large language models to infer context-sensitive semantic frames.

Method

Figure 2: Overview of the Semantic Valency Conflict (SVC) architecture.

We define Semantic Valency Conflict (SVC) as a condition where multiple interpretations of a lexeme activate incompatible semantic valency profiles in a given environment. The SVC method aims to detect ambiguity arising from lexical polysemy and underspecified argument structures.

Ambiguity arises when a single lexeme activates multiple incompatible cognitive frames F. Each stage of the proposed architecture mirrors the cognitive process of frame activation and conflict resolution:

  • Extraction of actants and alignment with environmental constraints E.
  • Dynamic frame induction using large language models (LLMs).
  • Evaluation of activated cognitive frames {Fk} and their valency profiles {Vk}.

A predicate is flagged as ambiguous when two interpretations k ≠ m produce semantically incompatible valency profiles:

Vk ≁E Vm

Alternatively, if only one frame Fk is active but multiple variants {vi} exist within it:

|{vi}| > 1

This represents variation within a single event schema (e.g. “olive oil” vs “sunflower oil”) rather than true semantic conflict. Hence, SVC distinguishes genuine ambiguity from contextual variability, forming a robust basis for disambiguation.

Experimental Evaluation

Table 1: Performance of our method (SVC) in comparison to baselines on the AmbiK dataset. Comparison across models (GPT, Mistral, Qwen) and methods. Metrics include Correct Help Rate (CHR), Help Rate (HR), and Intent Alignment (IA), each broken down by instruction type: Unambiguous (UN), Preference (PR), Common Sense (CS), and Safety (S)

Metrics

HR (Help Rate): frequency of the module requesting clarification.
CHR (Correct Help Rate): appropriateness of the help request based on actual ambiguity.
IA (Intent Alignment): similarity between predicted and ground-truth intents.
CHQ (Contextual Help Quality): harmonic mean of IA and CHR:

CHQ = 2 · IA · CHR / (IA + CHR + ε)

Table 2: Contextual Help Quality (CHQ↑) of SVC and baseline systems on the AmbiK and IntroPlan datasets. CHQ reflects the usefulness of contextual suggestions in supporting user goals. Evaluation on Mistral. Best values are highlighted in bold.

Datasets

IntroPlan Mobile Manipulation: 600 short single-step tasks, scene descriptions, object lists. Assesses model sensitivity to varied phrasing but lacks detailed ambiguity definitions.

AmbiK: 2,000 annotated instructions covering semantic ambiguity types: user preferences, commonsense knowledge, safety considerations. Each task includes clarifying questions, answers, and execution plans.
